# 使用者指南（User Guide）

> Gesture Lab `v0.0.1`  
> 給一般使用者：如何安裝、錄手勢、觸發反應、跨裝置同步。

## 1. 這是什麼？

Gesture Lab 讓你可以：

1. 用鏡頭**錄下手勢**（單手或雙手）
2. 設定觸發後的**反應**（廣東話朗讀，或播放音樂／音訊）
3. 之後再做同樣手勢，就會自動觸發
4. （建議）用**同步碼**把資料存進資料庫，換手機／電腦也能用

## 2. 環境需求

- 支援的瀏覽器：Chrome、Edge、Safari（建議最新版）
- 需要**相機權限**
- 裝置：手機或筆電皆可
- 朗讀廣東話：系統需有「中文（香港）」粵語語音（macOS／iOS 通常較完整）

> 相機只在安全連線下可用：`localhost` 或 HTTPS。用區網 IP 開網頁時，部分手機會擋相機。

## 3. 最快開始（Docker）

若你已有 Docker：

```bash
git clone https://github.com/wongkino/AR-App.git
cd AR-App
docker compose up --build -d
```

瀏覽器開啟：**http://localhost:8080/**

停止：

```bash
docker compose down
```

### 使用已建好的映像（GHCR）

```bash
docker compose pull
docker compose up -d
```

映像：

- `ghcr.io/wongkino/ar-app:0.0.1`（或 `:latest`）— 前端與 API 同一服務

若無法拉取，請先登入 GHCR，或到 GitHub Packages 將套件設為 Public。

## 4. 介面說明

畫面分為兩區：

| 區域 | 內容 |
|------|------|
| 左側／上方 | 相機畫面 + 手部骨架（左綠右橘） |
| 右側／下方 | 雲端同步、錄製、監聽、已儲存列表 |

狀態提示：

- **等待手部／偵測到 1～2 隻手**
- **錄製中 · N 幀** / **監聽中**
- 同步狀態：已連線資料庫／同步中／同步失敗／離線

## 5. 建議流程（第一次）

### 步驟 A：一般使用者（只監聽）

1. 開啟頁面（會自動從資料庫載入手勢）
2. 開啟相機 → **開始監聽**
3. 做出已錄好的手勢即可觸發反應

### 步驟 B：管理員（錄製／儲存）

1. 在「管理密碼」輸入密碼 → **解鎖編輯**
2. 開啟相機 → 錄製 → 停止 → **儲存手勢**（自動寫入資料庫）
3. 可改反應、刪除；完成後可按 **鎖定編輯**

預設管理密碼：`gesture-admin`（可用環境變數 `ADMIN_PASSWORD` 修改）

## 6. 管理已儲存手勢

在「已儲存」列表：

| 按鈕 | 作用 |
|------|------|
| **改反應** | 改名稱、朗讀文字或音訊網址 |
| **試播** | 不需做手勢，直接聽反應 |
| **刪除** | 刪除該手勢（有同步時會一併從資料庫移除） |

## 7. 跨裝置

1. 裝置 A 建立同步碼並錄好手勢  
2. 裝置 B 開啟同一個網站（或同一套 Docker 服務）  
3. 輸入同步碼 → **加入同步**  
4. 應看到相同手勢列表；可按 **重新下載** 強制從資料庫拉最新  

**解除同步**：只斷開雲端綁定，本機快取仍會留著。

## 8. 常見問題

### 為什麼沒有廣東話？

檢查系統是否安裝粵語語音。macOS：系統設定 → 輔助使用 → 朗讀內容 → 系統語音 → 加入「中文（香港）」。

### 為什麼辨識不到／一直誤觸？

- 光線充足、手完整入鏡  
- 錄製與觸發時距離、速度盡量接近  
- 手勢再錄一次、動作更明確  
- 監聽時先停一下再做，避免與錄製姿態混淆  

### 手機連電腦 IP 無法開相機？

需要 HTTPS。可用隧道（如 Cloudflare Tunnel／ngrok）或只在本機 `localhost` 使用。

### 同步失敗？

確認 Docker 的 `api`、`db` 有在跑：`docker compose ps`，並訪問 `/api/health` 應回 `{"ok":true}`。

### 資料會上傳相機畫面嗎？

不會。上傳（若有同步）的是手部關鍵點軌跡與你設定的反應文字／音訊網址，不是錄影畫面。

## 9. 版本

目前文件對應版本見根目錄 `VERSION`。  
更新紀錄見 [CHANGELOG.md](../CHANGELOG.md)。
