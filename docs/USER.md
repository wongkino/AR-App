# 使用者指南（User Guide）

> Gesture Lab `v0.0.1`  
> 給一般使用者：如何安裝、錄手勢、觸發反應、跨裝置同步。

## 1. 這是什麼？

Gesture Lab 讓你可以：

1. 用鏡頭**錄下手勢**（單手或雙手）
2. 設定觸發後的**反應**（廣東話朗讀，或播放音樂／音訊）
3. 之後再做同樣手勢，就會自動觸發
4. （建議）用**同步碼**把資料存進資料庫，換手機／電腦也能用

## 2. 環境需求

- 支援的瀏覽器：Chrome、Edge、Safari（建議最新版）
- 需要**相機權限**
- 裝置：手機或筆電皆可
- 朗讀廣東話：系統需有「中文（香港）」粵語語音（macOS／iOS 通常較完整）

> 相機只在安全連線下可用：`localhost` 或 HTTPS。用區網 IP 開網頁時，部分手機會擋相機。

## 3. 最快開始（Docker）

若你已有 Docker：

```bash
git clone https://github.com/wongkino/AR-App.git
cd AR-App
docker compose up --build -d
```

瀏覽器開啟：**http://localhost:8080/**

停止：

```bash
docker compose down
```

### 使用已建好的映像（GHCR）

```bash
docker compose pull
docker compose up -d
```

映像：

- `ghcr.io/wongkino/ar-app-web:0.0.1`（或 `:latest`）
- `ghcr.io/wongkino/ar-app-api:0.0.1`（或 `:latest`）

若無法拉取，請先登入 GHCR，或到 GitHub Packages 將套件設為 Public。

## 4. 介面說明

畫面分為兩區：

| 區域 | 內容 |
|------|------|
| 左側／上方 | 相機畫面 + 手部骨架（左綠右橘） |
| 右側／下方 | 雲端同步、錄製、監聽、已儲存列表 |

狀態提示：

- **等待手部／偵測到 1～2 隻手**
- **錄製中 · N 幀** / **監聽中**
- 同步狀態：已連線資料庫／同步中／同步失敗／離線

## 5. 建議流程（第一次）

### 步驟 A：雲端同步（強烈建議）

1. 打開「雲端同步」
2. 按 **建立新同步碼**
3. 記下類似 `ABC-123-XYZ` 的碼
4. 換裝置時輸入同一組碼 → **加入同步**

沒有同步碼也可以用，但資料只在該瀏覽器本機，清資料或換裝置就沒了。

### 步驟 B：開啟相機

按 **開啟相機**，允許權限。把手放進畫面，應看到骨架。

### 步驟 C：錄製手勢

1. 填「名稱」（例如：揮手）
2. 選反應：
   - **朗讀廣東話**：輸入要讀的字
   - **播放音訊**：貼上 mp3／wav 網址
3. 按 **開始錄製**，做完整段動作
4. 按 **停止並預覽**
5. 按 **儲存手勢**

提示：動作約半秒以上較穩；雙手手勢請兩隻手都入鏡。

### 步驟 D：監聽觸發

1. 按 **開始監聽**
2. 重複剛才的手勢
3. 成功時會顯示「觸發：…」並播放反應

同一手勢短時間內不會連續狂觸發（有冷卻）。

## 6. 管理已儲存手勢

在「已儲存」列表：

| 按鈕 | 作用 |
|------|------|
| **改反應** | 改名稱、朗讀文字或音訊網址 |
| **試播** | 不需做手勢，直接聽反應 |
| **刪除** | 刪除該手勢（有同步時會一併從資料庫移除） |

## 7. 跨裝置

1. 裝置 A 建立同步碼並錄好手勢  
2. 裝置 B 開啟同一個網站（或同一套 Docker 服務）  
3. 輸入同步碼 → **加入同步**  
4. 應看到相同手勢列表；可按 **重新下載** 強制從資料庫拉最新  

**解除同步**：只斷開雲端綁定，本機快取仍會留著。

## 8. 常見問題

### 為什麼沒有廣東話？

檢查系統是否安裝粵語語音。macOS：系統設定 → 輔助使用 → 朗讀內容 → 系統語音 → 加入「中文（香港）」。

### 為什麼辨識不到／一直誤觸？

- 光線充足、手完整入鏡  
- 錄製與觸發時距離、速度盡量接近  
- 手勢再錄一次、動作更明確  
- 監聽時先停一下再做，避免與錄製姿態混淆  

### 手機連電腦 IP 無法開相機？

需要 HTTPS。可用隧道（如 Cloudflare Tunnel／ngrok）或只在本機 `localhost` 使用。

### 同步失敗？

確認 Docker 的 `api`、`db` 有在跑：`docker compose ps`，並訪問 `/api/health` 應回 `{"ok":true}`。

### 資料會上傳相機畫面嗎？

不會。上傳（若有同步）的是手部關鍵點軌跡與你設定的反應文字／音訊網址，不是錄影畫面。

## 9. 版本

目前文件對應版本見根目錄 `VERSION`。
